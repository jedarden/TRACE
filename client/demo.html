Integration Code

Add this to your website's <head> section:

<script src="https://your-domain.com/trace.js" data-collector="https://your-domain.com/collect"></script>
Configuration Options:
• data-collector: The collector endpoint URL
• data-debug: Set to "true" to enable console logging

Public API

TRACE exposes a global API for custom event tracking:

Custom Events

// Send a custom event TRACE.collect('custom', { action: 'button-click', value: 'signup' }); // Shorthand - defaults to type 'custom' TRACE.collect({ category: 'engagement', label: 'video-play' });

User Identification

// Identify a user with your own ID TRACE.identify('user-12345');

Manual Pageview

// For SPAs - trigger a pageview event TRACE.pageview();

Get IDs

// Get current session and user IDs console.log(TRACE.getSessionId()); console.log(TRACE.getUserId());

Reset Session

// Create a new session TRACE.reset();

Each threshold sends a scroll event with the current depth percentage.

Event Types

TRACE automatically sends these event types:

  • pageview - Initial page load and SPA navigation
  • dwell - Heartbeat every 30 seconds (configurable)
  • scroll - When scroll thresholds are reached
  • click - When links are clicked
  • identify - When user is identified
  • custom - Any custom events you send

Data Privacy

Privacy-First Design:
  • No third-party cookies
  • All data stored in localStorage (first-party only)
  • Session IDs are random UUIDs
  • No cross-site tracking without explicit link decoration
  • User IDs persist until manually changed

Browser Support

TRACE works in all modern browsers that support:

  • localStorage
  • sessionStorage
  • navigator.sendBeacon() or fetch API
  • URL and URLSearchParams APIs

IE11 Support: Falls back to fetch for sendBeacon. Requires polyfills for URL/URLSearchParams.
